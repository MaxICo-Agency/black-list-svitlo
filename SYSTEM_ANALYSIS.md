# Black List Світло парк: стан системи

## Що працює

- Пошук приймає номер із пробілами, дужками, дефісами, без `+38` або без `0`
  та нормалізує його до `+380XXXXXXXXX`.
- Один майстер може мати до 10 номерів, кілька послуг і кілька окремих відгуків.
- Головна, профіль, послуги, усі рекомендації, усі скарги, правила та privacy
  мають чисті URL і спільний адаптивний дизайн.
- Після пошуку результат оновлюється всередині центрального блока без
  автоматичного стрибка сторінки.
- На картках головної немає вигаданих аватарів. Реальні фотографії
  відображаються тільки всередині відгуку, до якого їх додали.
- У базі немає демо-майстрів, демо-відгуків або підставних фотографій.

## Заявка й модерація

1. Сайт відкриває `@bl_svitlopark_bot` з типом заявки та, за можливості,
   уже введеним номером.
2. Бот збирає контакти, послуги, текст, автора й фотографії в одному
   повідомленні, яке оновлюється на кожному кроці.
3. Приватна група модерації отримує компактну картку та оригінальні фото.
4. Модератор схвалює запис, обирає готову причину або вводить власну.
5. Автор отримує рішення, причину відхилення та контакт адміністрації.
6. Схвалений запис одразу з'являється в API, на сайті й у публічному каналі.

Головне меню бота містить тільки пошук, рекомендацію, скаргу, власні заявки й
контакт адміністрації. У `Моїх заявках` автор може змінити текст уже
розглянутого відгуку або його тип. Нова версія проходить окрему модерацію;
попередня не зникає з бази до схвалення заміни.

Картки результатів, модерації та каналу побудовані на Bot API 10.2 Rich
Messages. Службові поля приховані в нативному блоці деталей, текст відгуку
показаний цитатою, а головні дії мають семантичні кольори. Короткі запитання
бота використовують regular HTML, візуальний прогрес і коротке резюме вже
введених даних. Усі динамічні значення екрануються, а при помилці Rich Messages бот
автоматично переходить на сумісний HTML-варіант.

Публічна скарга завжди позначається як користувацький досвід, а не як
встановлений платформою факт. Автоматично витягнуті з будинкового чату записи
мають примітку `* Дані витягнуто автоматизовано з чату будинку.`

## Сховище

- текстові заявки й рішення: серверні JSONL-файли у persistent runtime volume;
- фото: Telegram file storage, доступне на сайті тільки після схвалення;
- нормалізована Google Sheet: приватний адміністративний знімок;
- `LiveDirectory`: автоматичне дзеркало актуального публічного API.

## Історичний імпорт

Read-only скан папки `🏠ЖК🏠` за 360 днів переглянув 18 701 повідомлення.
Вісім раніше перевірених рекомендацій залишені в робочій базі. Нові кандидати
не публікуються автоматично: прямі рекомендації та три скарги, привʼязані до
конкретних номерів, надходять у приватну групу разом із source reference.
Неоднозначні фрагменти лишаються на ручну перевірку, а самореклама не рахується
рекомендацією мешканця.

## Відомі межі

- Bot API не дозволяє знайти приватний профіль, username або аватар людини
  лише за номером телефону. Імпорт номера через особистий Telegram-акаунт
  змінює список контактів і все одно залежить від privacy settings, тому сервіс
  цього не робить. Уже відомий username автоматично береться з власної бази.
- Сервіс не визначає правдивість претензії автоматично; остаточне рішення
  приймає модератор після перегляду контексту й доказів.
- Google Sheets є адміністративним дзеркалом. Джерелом оперативного стану для
  сайту є схвалена серверна runtime-база, щоб зміни з Telegram з'являлися одразу.
